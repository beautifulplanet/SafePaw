# Log sanitization audit (O4)

**Status:** Audit complete  
**Date:** 2026-03-13  
**Scope:** Wizard and Gateway log sites; no tokens, passwords, or PII in logs.

---

## 1. Policy

- **Never log:** Passwords, API keys, session tokens, gateway auth tokens, TOTP secrets, or any secret from `.env`.
- **Never log:** Full request/response bodies (may contain user messages or tokens). Log metadata only: path, length, risk level, request_id.
- **Always sanitize:** User-controlled or network-derived strings (IP, path, Origin, X-Request-ID) with a control-character strip before interpolation to prevent log injection (forged newlines/entries).
- **Audit log:** Records who (IP), what (action), outcome; may record key names (e.g. `keys_changed`) but never secret values.

| `internal/audit/audit.go` | Actor (IP), action, resource, outcome, details (e.g. `keys_changed=WIZARD_ADMIN_PASSWORD`, `subject`, `scope`) | No | Details are key names or non-secret identifiers. |

**Sanitization:** `sanitizeLog(s string)` in handler.go strips control characters (0x00–0x1F, 0x7F) from IP and any user-derived string before logging (log injection prevention).

## 4. Accepted exceptions

| Exception | Reason |
|-----------|--------|
| **Wizard first-run admin password** | Printed once to stdout when no `WIZARD_ADMIN_PASSWORD` is set so the operator can set `.env`. Documented in SECURITY.md and README. Not logged again. |

---

## 5. What we never do

- Log request or response body content (only length, risk level, or pattern names).
- Log `AUTH_SECRET`, `WIZARD_*_PASSWORD`, `WIZARD_TOTP_SECRET`, `REDIS_PASSWORD`, `POSTGRES_PASSWORD`, or any `.env` secret.
- Log issued gateway tokens or session cookie values.
- Put unsanitized user-controlled or network-derived strings into log format strings (prevents log injection).

---

## 6. Maintenance

- **New log sites:** Before adding `log.Printf`/`Logger` calls that include request data, use `sanitizeLog` (wizard) or `SanitizeLogValue` (gateway) for any string that may contain control characters or user input.
- **Audit details:** When adding `audit.Log(..., details map[string]string)`, ensure `details` contains only key names or non-secret identifiers (e.g. subject, scope), never secret values.
- **CI:** No automated log-scan yet; this doc is the checklist. Consider a grep or custom linter for `log.Printf.*%s.*password|token|secret` in a later iteration.
