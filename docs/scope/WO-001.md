# WO-001: SafePaw Control, Logging, and Graceful Shutdown

**Date:** 2026-03-10
**SOW Ref:** SOW-001 / SCOPE-IMPROVEMENTS T2, BREAK_LIST #50
**Phase:** Operational Reliability
**Status:** IN PROGRESS

---

## 1. Objective

Give SafePaw a proper "off button," persistent runtime logging with a live viewer,
and graceful shutdown handling at all layers. A non-technical user should be able to
start, monitor, and stop the entire stack from their desktop.

## 2. Scope

### Deliverable 1: Start / Stop / Status Scripts
- [ ] Create `stop.bat` — clean shutdown of all services
- [ ] Create `stop.sh` — Linux equivalent
- [ ] Create `status.bat` — show running/stopped state of each service
- [ ] Create `status.sh` — Linux equivalent
- [ ] Create `create-shortcuts.bat` — generates desktop shortcuts for Start/Stop/Status

### Deliverable 2: Per-Session Logging + Live Viewer (2 windows)
- [ ] Create `logs/` directory convention
- [ ] Modify `start.bat` to start log aggregation to `logs/session-YYYY-MM-DD-HHMMSS.txt`
- [ ] Modify `start.bat` to open Window 1: process/container status (refreshing)
- [ ] Modify `start.bat` to open Window 2: live log viewer (tailing session log)
- [ ] Modify `stop.bat` to close viewer windows on shutdown
- [ ] Modify `start.sh` for Linux equivalents

### Deliverable 3: Graceful Shutdown in Go Services
- [ ] Add SIGTERM/SIGINT handler to `services/gateway/main.go`
- [ ] Add `server.Shutdown()` with timeout for connection draining
- [ ] Close Redis connection pool on shutdown
- [ ] Flush audit ledger on shutdown
- [ ] Add SIGTERM/SIGINT handler to wizard entry point
- [ ] Add `server.Shutdown()` to wizard
- [ ] Close Postgres/Redis connections on wizard shutdown
- [ ] Test: `docker compose down` triggers graceful shutdown log messages

## 3. Acceptance Criteria

- [ ] `stop.bat` stops all containers and closes viewer windows
- [ ] `status.bat` shows per-service health in a readable format
- [ ] Session log file persists after shutdown with complete timeline
- [ ] Window 1 shows container status and refreshes while running
- [ ] Window 2 shows live log output from all services
- [ ] Both windows auto-close on `stop.bat`
- [ ] Gateway logs "shutting down" and "shutdown complete" on SIGTERM
- [ ] Wizard logs "shutting down" and "shutdown complete" on SIGTERM
- [ ] In-flight HTTP requests complete before process exits (15s timeout)
